Efficient Algorithms

Imagine you come across the following task: add all numbers from 1 to 1,000,000...

So how do we do this? One way would be to use a loop:

const number = 1000000;
let result = 0;
for (let i = 1; i <= number; i++) {
result += i;
}
console.log(result);

The result is 500,000,500,000.

So far, so good. One wouldn't probably worry about computer ressources given the raw CPU power nowadays. At the other hand, what if you could get the result without having to loop a million times?

You may already have heard of Carl Friedrich Gauß. He found a useful formula to calculate this:

n * (n + 1) / 2

Which brings us back to our previous task but instead the loop we use this formula:

const number = 1000000;
const result = number * (number + 1) / 2;
console.log(result);

The result is, no surprise, 500,000,500,000 but archieved with way less effort.

As one can imagine, this could have an big impact in huge calculation heavy applications and it is always worth to make your code as efficient and light on ressources as possible.
